Linux File System Hieracrchy

  1. Linux file system has tree like structure
  2. In Linux drive concept is not available
    like c drive , d drive
  3. In Linux file system Hieracrchy top most directory is /
  4. / is considered as root directory or parent directory
  5. Under / we have sub-directory
    like bin,sbin,etc,user,home,root,opt,mnt,media,proc dev sys,lib,var,boot,tmp.


1.bin -- bin means binary

    -- binary executable files are stored inside bin
    -- Actually commands related binaary executable files are available in bin directory


2.sbin -- sbin stands for system bin

    -- super user realted binary executable files are available in sbin directory
    -- example: 1.Disk Partitioning
                       2.Network Managment

3.etc directory

    -- This directory contains all system configuration related files.
    -- These configuration files are used oprating system itself.
    -- All user realted information is available inside
                      /etc/passwd file
    -- All group related information is available inside
                      /etc/group file

4.dev directory-- dev means device

    -- In Linux everthing is treated as a file
    -- Every device in Linux is treated as a file
    -- All device related files will be available inside de directory
    -- By using device files we can communicate with devices

5.boot directory

    -- Files which are required to boot or to start Linux oprating system are available inside boot directory
    -- All these files are internally used by operating system

6.lib directory

    -- lib stand for library
    -- These library files are required by our command or application

7.tmp Directory

    -- tmp means temporary
    -- If any file or directory is required to create for temporary purpose then those files or directory are created inside tmp     directory
    -- Files or directory available inside tmp directory be deleted automatically at the time of system shutdown

8. mnt directory

    -- mnt means mounting
    -- Attaching external device to already existing file is mounting
    -- In old operating system mounting is done manually
    -- In newer operating system mounting happens automatically
    -- mnt directory contains files which are available just because of manual mounting

9. media directory

    -- files of automatic mounting will be placed in media directory
    -- Note : Bydefault media directory does not contains any audio, video or images

10 opt directory

    -- opt means optional
    -- All third party installation files are available in opt directory
    -- If we install google chrome then all installation files will be place inside opt directory

11. var directory

    -- var means variable data
    -- The data which is keep on changing such type of variable data will be stored inside var directory
    -- Example log file

12. usr directory

    -- usr means user
    -- this directory contains all user related softwares
    -- Example : All vendor releted softwares
    -- Here vendor means a person or organization providing softwares on non-profitable form.
           Example Mysql
    -- If user install venfor specific softwares then installed softwares goes inside usr directory

13. home directory

    -- For every user a seperate directory is created to hold or to maintain user specific data
    -- All these user directories will be stored inside home directory
    -- Example 1: Suppose I have created user named as RRD, Ravi and MELISA
                /home/rrd --- This is rrd home directory
                  /home/ravi --- this is ravi home directory
                  /home/melisa --- this is melisa home directory
