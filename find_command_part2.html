find command
Syntax:
find location type option file_or_dir_name

#Find an empty file or an empty directory
Example 1: find all empty files from current location
Syntax: find . -type f -empty

Example 2: Find all empty files from current location where name start with A
Syntax: find . -type f -empty -name 'A*'

Example 3: Find all empty directory from current location
Syntax: find . -type d -empty


# Find file or directory on the basis of size
- To specify size we use -size option1
- To specify size value use below symbols
  1. M-> MB
  2. G-> GB
  3. k -> KB
  4. c -> byte
  5. + -> Greater than
  6. - (minus) -> less than

Example 1: find all files from current location where size is 2kb
Syntax: find . -type f -size 2k (use small k)

Example 2: find all files whose size is less than 2 byte
Syntax: find . -type f -size 2c

Example 3: find all files whose size is greater than 2 byte but less than 4 byte
Syntax: find . -type f -size +2c -size -4

Example 4: find all files from current location where size is 2 byte or 4 byte
Syntax: find . -type f -size 2c -o -size 4c

Note : To specify or conditio we need to use -o option

# find file on the basis of created time, modified time and access time

Example : find all files from current location which are created 10 days back
Syntax: find . -type f -ctime -10

Example : find all files from current location which are modified 10 days back
Syntax: find . -type f -mtime -10

Example: find all files current location which are accessed 10 days back
Syntax: find . -type f -atime -10

Example: find all files from current location which are created 10 min back
Syntax: find . -type f -cmin -10

Example: find all directory from current location which are modified 10 min back
Syntax: find . -type f -mmin -10

Example: find all directory from current location which are accessed 10 min back
Syntax: find . -type d -amin -10



#Find command with -exec option
- Ones files is found, If we want to perform some operation on that result like
copy a file, delete a file , move from one location to another
then use -exec (execute) option with find command

Example: find all log files from current location and mv to Directory ABC
Syntax: find all log files from current location
                find . -type f -name '*.log' -exec mv {} ABC \;

IMP points
  1. - There is space between -exec option and command name
  2. - There is space between command name (mv) and curly bracket
  3. - There is space between Directory name (ABC) and curly bracket
  4. - There is space between Directory name (ABC) and backword slash
  5. - There is no space between backword slash and semicolon
Example: Find all log files from current location and delete those files
Syntax: find . -type f -name '*.log' -exec rm -f {} \;
Note: If we want delete a file then use rm command and its syntax is
           rm -f file_name
           -f means revome forcefully
Note: If we want remove non-empty and empty directory then also we can use rm command
Syntax: rm -rf directory_name
           Here -r stands for recursively
           f stands for forcefully

Example: find cutdemo.txt file from current working path and print its content on screen
Syntax: find . -type f -name cutdemo.txt -exec cat -n {} \;


# locate command
- locate command is also used to find files
- find command searches file in a linux file system
- locate command searches file in linux database

- syntax
           locate file_dir_name

-Note: in locate command no need to spacify search location
   because it always searches a file in database
- by default locate command is not available but we can install maunally as below
   sudo apt install plocate or sudo apt install mlocate


Q. what is difference find command and locate command

find:
  1. find command always searches a file in file system
  2. find command is more powerfull than locate
  3. in find command we can find file on the basis of name,
        size,user,group ,perm, time
  4. find command is slower than locate
  5. find command always give accurate result
  6. we can use -exec option to perform operation on result


locate:
  1. locate command always searches a file in database
  2. locate command is less powerfull than find
  3. In locate command we can find file on the basis on name
  4. locate command is faster than find
  5. locate command may or may not give accurate result
  6. We can't use -exec option to perform operation on result
Q. Why find command is more powerfull than locate ?
Ans: While finding a file using find command we can use different criteria
         like name, size,perm, time but in locate command we can find file only name basis


Q. Why locate command is faster than find?
Ans: locate command searches a file at only one place i.e database but find command
        searches file in file system


Q. locate command may or may not give accurate result, Why?
-- 1. locate command searches file in a database
    2. bydefault this database get updated ones a day
    3. suppose after database update you have created some files
        then those file entry will not exist in database.so that
        locate command will not produce accurate result


how to update database manually
Syntax: sudo updatedb
