#piping concept


Q. What is mean by piping? - passing output of one command to another command using pipe operator is nothing but piping
- Symbol of pipe operator is |
- syntax
     command1| command2 | command3 | commandN
    Here command 1 get executed and its output is passed to command 2 as input
    command 2 get executed and its output is passed to command 3 as input in this
    way command get executed and output passed to another command
    last command output get display on the screen or we can redirect into a file


Example 1: count files and directory present at current location
Ans: ls | wc -w

Example 2: cut first five character from line number 5 of cutdemo.txt file

    Step 1: cut first five character
    cut -c 1-5

    Step2: print line number 5
    sed -n '5p'

    Step3: sed -n '5p' cutdemo.txt | cut -c 1-5

#grep command
- grep stands for Globle Regular Expreesion print
- use: This command is used to find or search particular pattern in
single or multiple files
- bydefault grep command search pattern by considering case
- syntax:
         grep option pattern file_name


Example 1: find "root" word in /etc/passwd file
     grep " root" /etc/passwd
                      or
     cat -n /etc/passwd | grep "root"

Example 2: find "root" word in /etc/passwd by ignoring case
     grep -i "Root" /etc/passwd
     cat -n /etc/passwd | grep -i " root"

Example 3: find "sys" word from top 5 lines of /etc/passwd file
     cat -n /etc/passwd | head -5 | grep -i "sys "
    


Some Regular Expreesion 1. To match any character use *
2. To match single character use ?
3. To provide range use []
    Example : count only.txt file from current
4. [abc] -- either a or b or c
5. [aeiou] -- either a or e or i or o or u
6. [AEIOU] -- either A or E or I or O or U
7. [a-z] -- any lower case character
8. [A-Z] -- any upper case character
9. [0-9] -- any digit from 0 to 9
10. ^ (cap symbol) -- start of line or beginig of lin either
11. $ -- end of line


Example :: count only files from current working path
Syntax:   ls -l | grep '^-' | wc -l


Example : : count directory from current working path
Syntax:   ls -l | grep '^d' | wc -l


Example :: count only link files from current working path
Syntax:   ls -l | grep '^l' | wc -last


Example : : count only text file from current location
    - ls -l | awk '{print $9}' | grep '.txt ' | wc -l
    - ls | grep '.txt' | wc -l
    - find . -type f -name '*.txt' | wc -last
    -w option of grep command
    - To extract match


Example :: find roo from /etc/passwd file
Syntax:   cat -n /etc/passwd | grep " roo" -> without -w option
Syntax:   cat -n /etc/passwd | grep -w "roo" -> with -w option


# -A -B and -C option
    -A: Print line after matched pattern -B: Print lines before matched pattern -C: Print lines After as well as before matched pattern


Example :: Find "sys" word from /etc/passwd file and print 3 lines
after match pattern Syntax:   cat -n /etc/passwd | grep -A3 "sys"


Example :: count only blank lines
Step 1: Print only blank lines Syntax:   grep '^$' cutdemo.txt

Step : count blank lines Syntax:   grep '^$' cutdemo.txt | wc -l

Step 3: Display file content by ignoring blank lines Syntax:   grep -v '^$' cutdemo.txt

-v is for inversion
inversion means change or reverse the output



Example :: Delete blank lines from a file
Syntax:   grep -v '^$' cutdemo.txt > file_name
